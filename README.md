## 开发目的
1. 用户需求层面：
   市面上多数笔记类APP存在「功能冗余」「广告多」「依赖网络」「隐私风险」等问题，普通用户仅需一款「轻量化、无广告、纯本地、操作简单」的基础备忘录工具，满足日常记录、快速查找、数据安全的核心需求。
2. 技术实践层面：
   作为Android开发实践项目，可完整覆盖「本地数据持久化」「UI交互设计」「异步处理」「版本兼容」等核心开发场景，验证现代Android开发技术栈（Kotlin+Jetpack）的落地能力。
3. 场景适配层面：
   学生、职场人士等群体有「离线记录」需求（如会议纪要、学习笔记、待办事项），无需云同步的纯本地备忘录能避免网络依赖，同时保障数据隐私不泄露。

   开发一款「极简、稳定、安全」的本地备忘录应用，解决用户“轻量记录、快速查找、数据不丢失”的核心痛点，无需复杂注册/登录，打开即可使用。

## 项目实现的核心功能

备忘录管理
•添加备忘录：用户可以创建新的备忘录，输入标题、内容、选择分类，并设置提醒时间。
•编辑备忘录：用户可以修改已存在的备忘录信息，包括标题、内容、分类和提醒时间。
•删除备忘录：用户可以删除不需要的备忘录。
•查询备忘录：支持通过标题关键词模糊查询备忘录。
•统计备忘录：能够统计当前所有备忘录的总数。

提醒功能
•用户可以为备忘录设置提醒时间，提醒时间可以精确到分钟。
•到达提醒时间时，应用可以通过震动或播放音频的方式提醒用户。

•分类管理
•备忘录可以按类别进行分类管理，如工作、生活、学习等。
•用户可以在添加或编辑备忘录时选择相应的分类。

•界面设计
•主界面：显示日历视图和备忘录列表。日历视图左侧添加一列显示教学安排的周数。
•添加/编辑界面：提供输入框用于输入备忘录的标题、内容，下拉框用于选择分类，时间选择器用于设置提醒时间，以及保存按钮完成操作。

## 功能模块设计

界面展示模块
•主界面：包含日历组件（预留显示教学安排周数功能）、备忘录列表展示区域和操作按钮。
•添加 / 编辑备忘录界面：提供输入框、下拉框、时间选择器和保存按钮，用于输入和保存备忘信息。

数据存储模块
•利用 SQLite 数据库存储备忘录数据，通过DatabaseHelper创建和管理数据库，MemoDao实现数据的增删改查操作。

提醒功能模块
•由ReminderUtil设置提醒，ReminderReceiver接收广播并处理提醒事件，实现震动或音频提醒。

事件分类管理模块
•在界面提供分类选择和展示，在数据存储和查询中支持按分类操作，方便用户分类管理备忘录。

数据交互与界面更新模块
•通过MemoAdapter实现数据与界面的绑定，在数据变化时及时更新界面显示，确保数据一致性。
